Lists are made up of multiple rows of items which can contain text, buttons, toggles, icons, thumbnails, and much more. Lists generally contain items with similar data content, such as images and text.

Lists support several interactions including swiping items to reveal options, dragging to reorder items within the list, and deleting items.

Virtual Scroll

The virtual scroll should be mostly invisible to the user, but behind the scenes, only a small subset of the total number of list items are actually rendered on the page. Items will dynamically be added/removed from the top and bottom of the list as the user scrolls up or down. This can have a very positive impact on performance - especially on long lists.

The virtual scroll implementation is based on the ngx-ui-scroll library.

Load on demand with Virtual Scroll

Virtual scroll is particularly useful in combination with load-on-demand lists. When new items are continually requested and added to the existing page, it can result in a very large and poor performing document. Virtual scroll fixes this by preserving the loaded items in memory, but only rendering the necessary items on the page.
