Action-sheet reveals a sheet with a set of options when tapping the trigger button. The sheet behaves differently depending on device. On a touch device the sheet will slide up from the bottom of the screen, whereas on larger screens, the sheet is displayed next to the trigger button.

Default

When no custom button is provided between the <kirby-action-sheet> tags, a default icon-button is displayed.

Custom Button

If the default button icon doesn't fit the context, you can configure a custom button icon through the iconName property. Allowed icon names can be found here: Icons.

To add text to the trigger button, simply add a buttonText.

Disabled

Open Action Sheet Programatically

To show the action sheet, create an ActionSheetConfig and pass it to modalController.showActionSheet. Each item must to be of type ActionSheetItem.

Caveat: Showing the action-sheet in this way will not ascociate the sheet with the trigger element. It will always slide up from the bottom of the screen - even on non-touch devices.

const config: ActionSheetConfig={{ '{' }} header: 'Your action sheet header', subheader: 'Your action sheet sub-header', items: [ {{ '{' }} id: '1', text: 'Option 1' {{ '}' }}, {{ '{' }} id: '2', text: 'Option 2' {{ '}' }}, {{ '{' }} id: '3', text: 'Option 3' {{ '}' }}, ], cancelButtonText: 'Custom cancel', {{ '}' }}; this.modalController.showActionSheet(config); (Optional) if you need to obtain data back from the action sheet, you can pass a callback function: this.modalController.showActionSheet(config, onItemSelect); where onItemSelect can be e.g.: onItemSelect(item: ActionSheetItem) {{ '{' }}...{{ '}' }} We can use modalController.hideTopmost to hide (and destroy its component) the topmost action sheet. We can pass an optional result parameter to it, which is then going to be available inside the callback
